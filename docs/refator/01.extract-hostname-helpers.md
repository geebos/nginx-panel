# 01. Extract hostname normalize/compare helpers

## 发现的问题

多处 worker 代码复制了相同的多 hostname 归一化与集合比较逻辑：小写、去掉尾点、去重、排序，再用 `JSON.stringify` 比较。

## 问题证据和影响范围

相同函数体出现在：

- `src/worker/lib/acme/renewal.ts` — `normalized` + `sameHostnames`
- `src/worker/lib/acme/activation.ts` — 同上
- `src/worker/lib/acme/certificate-store.ts` — `normalized`
- `src/worker/lib/manager/service.ts` — `sameHostnameSet`（内联 normalize）

影响：SAN drift / manager hostname rebind 等关键路径若某处被改、别处未改，会产生行为漂移。

## 原始设计

各模块私有 helper，逻辑字节级一致，无共享模块。

## 最小修改方案

新增纯模块 `src/worker/lib/hostnames.ts`，导出：

- `normalizeHostnames`
- `sameHostnames`

四处改为 import；不碰单值 normalize、不碰 `assertHostnamesMutable` 的较弱比较。

## Design Challenger 结论

- **verdict**: APPROVE
- risks: 勿误统一 `assertHostnamesMutable`；保持 default string sort 与 JSON.stringify 比较
- simpler-alternative: 仅 acme 内抽取会留下 manager 第二真相源，否决
- required-tests: normalize / sameHostnames 单元测试（大小写、尾点、去重、排序、顺序无关）

## TDD 过程

1. 先写 `src/worker/lib/hostnames.test.ts`（import 缺失 → 失败）
2. 实现 `hostnames.ts` 使单元测试通过
3. 替换 4 处调用并跑 renewal/activation 回归

## 最终修改

- 新增 `src/worker/lib/hostnames.ts`
- 新增 `src/worker/lib/hostnames.test.ts`
- 更新 renewal / activation / certificate-store / manager/service

## Code Reviewer 结论

**APPROVED** — 无 BLOCKING；行为与既有实现一致，diff 小且可审。
