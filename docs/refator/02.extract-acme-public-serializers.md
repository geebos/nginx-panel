# 02. Extract ACME public API serializers

## 发现的问题

`publicOrder` / `publicChallenge` / `publicCertificate` 在 domain 证书路由与 manager 证书服务中字节级重复。

## 问题证据和影响范围

- `src/worker/routes/certificates.ts`
- `src/worker/lib/manager/certificate.ts`

三者实现经对比 `equal? True`。影响 API 响应形状（隐藏 idempotencyKey、dns 值裁剪、证书路径不外泄）。

## 原始设计

两文件各维护一份私有 serializer。

## 最小修改方案

抽取到 `src/worker/lib/acme/public.ts`；两处 import。不合并 create-order 流程（domain vs manager 有意分叉）。

## Design Challenger 结论

- **verdict**: APPROVE
- risks: 类型收敛时勿改行为；保持原样搬迁
- simpler-alternative: 无更简正确方案
- required-tests: 三个 pure serializer 单测

## TDD 过程

1. 写 `public.test.ts` + 实现 `public.ts`
2. 删除两处本地副本并改 import
3. 跑 public + certificates 测试与 typecheck/lint

## 最终修改

- 新增 `src/worker/lib/acme/public.ts`、`public.test.ts`
- 更新 `routes/certificates.ts`、`manager/certificate.ts`

## Code Reviewer 结论

**APPROVED** — 无 BLOCKING。NON_BLOCKING：测试 fixture 部分字段名与 schema 不完全一致（`as never`），不影响本轮行为验证。

## 验证结果

- public + certificates tests → 10 pass
- typecheck / lint → clean

## 后续候选优化

- Split one high-complexity pure helper from deployment/runner（若安全）
- Address empty catch in logs/parser（若可行为保持）
