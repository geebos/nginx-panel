# 03. Extract master key load + HKDF helpers

## 发现的问题

Master key 读取与 HKDF 派生逻辑在 auth、Cloudflare credentials、log cursor 三处重复，且错误处理已出现分叉。

## 问题证据和影响范围

- `src/worker/lib/auth.ts` — BusinessError 包装 + cache + `auth-attempts-v1`
- `src/worker/lib/cloudflare/credentials.ts` — BusinessError 包装 + `cloudflare-credentials-v1`
- `src/worker/lib/logs/cursor.ts` — 原错 rethrow + cache + `log-cursor-v1`

## 原始设计

各模块私有 `readFile` + 相同 dev fallback + 内联 `hkdfSync`。

## 最小修改方案

`src/worker/lib/master-key.ts`：

- `loadMasterKey()`：共享读文件 / dev 回退；非 dev 原样 rethrow
- `deriveManagerKey()`：共享 salt `nginx-domain-manager`

BusinessError 仍由 auth/credentials 调用点包装。路径在调用时解析。

## Design Challenger 结论

- **verdict**: APPROVE
- risks: 包装策略必须对齐；HKDF 参数不可漂
- simpler-alternative: 仅抽 load 亦可，但 derive 锁 salt 仍值得
- required-tests: load 成功/失败/dev 回退 + derive 确定性；credentials/auth 回归

## TDD 过程

1. 写 master-key 测试与实现
2. 改三处 call site
3. 修测试：路径改为 call-time 解析（便于 env stub）

## 最终修改

- 新增 `master-key.ts` / `master-key.test.ts`
- 更新 auth / cloudflare credentials / logs cursor

## Code Reviewer 结论

**APPROVED** — 无 BLOCKING。NON_BLOCKING：production BusinessError 包装测试可补；auth/cursor 进程内 cache 为既有行为。
