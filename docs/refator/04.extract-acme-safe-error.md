# 04. Extract ACME safe error message helper

## 发现的问题

activation 与 scheduler 各有一份 URL 脱敏 + 500 截断的 `safeError`。

## 问题证据和影响范围

- `src/worker/lib/acme/activation.ts` — fallback `"Certificate activation creation failed"`, placeholder `[URL]`
- `src/worker/lib/acme/scheduler.ts` — fallback `"ACME order processing failed"`, placeholder `[ACME URL]`
- `cloudflare-cleanup.ts` 仅截断、无 URL 脱敏（本轮不动）

## 原始设计

文件内私有 helper，逻辑相同、参数不同。

## 最小修改方案

`safeErrorMessage(error, fallback, urlPlaceholder = "[URL]")` 于 `src/worker/lib/acme/safe-error.ts`。

## Design Challenger 结论

- **verdict**: APPROVE（须使用 scheduler 精确 fallback 文案）
- risks: 错误 fallback 字符串会改变 DB 存档文案
- simpler-alternative: 无更好最小方案
- required-tests: passthrough / fallback / URL redaction / custom placeholder / truncate 500

## TDD 过程

1. 写 safe-error 测试与实现
2. 替换 activation + scheduler 调用
3. 跑 safe-error / activation / scheduler 测试

## 最终修改

- 新增 `safe-error.ts` / `safe-error.test.ts`
- 更新 activation.ts、scheduler.ts

## Code Reviewer 结论

**APPROVED** — 无 BLOCKING。

## 验证结果

- 16 tests pass（safe-error + activation + scheduler）
- typecheck / lint clean

## 后续候选优化

- Reduce nested ternaries in a high-traffic page helper
- Split pure helper from deployment/runner（若安全）
- Optionally unify cloudflare-cleanup truncation with redaction（行为变更，需单独评审）
