# 07. Extract recheckable ACME order statuses

## 本轮过程

1. 候选 manager domain insert factory → Design Challenger **REJECT**（太小、无缺陷证据、placement 耦合）
2. 改选 recheck 资格状态列表抽取 → **APPROVE**

## 发现的问题

domain 与 manager 的 manual recheck 使用相同三态字面量：
`waiting_http`, `waiting_dns`, `validating`

该集合是 `activeOrderStatuses` 的真子集（刻意不含 `preparing` / `validated` / `downloading`）。

## 方案

在 `order-status.ts` 增加 `recheckableOrderStatuses`，两处 recheck 改为引用。

## Design Challenger

- manager insert: REJECT
- recheckable statuses: APPROVE
