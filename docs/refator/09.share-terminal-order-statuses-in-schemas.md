# 09. Share terminalOrderStatuses via shared schemas

## 发现的问题

前端 3 处本地 `terminalOrders` 与 worker `terminalOrderStatuses` 内容相同，但 FE 不能 import worker，造成跨层漂移。

## 方案

- 在 `src/shared/schemas/certificate.ts` 导出 `terminalOrderStatuses`
- worker `order-status.ts` re-export，保持既有 worker import 路径
- 前端 SSL 相关页面/表单改为从 `@/shared/schemas` 引用
- active / recheckable 仍留在 worker（仅 worker 使用）

## Design Challenger

**APPROVE**（prefer 放 certificate.ts、同名导出）
