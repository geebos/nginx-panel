# 24. Extract certificateDisplayStatus

## 发现的问题

证书列表页的 badge / filter / summary 依赖同一套未测试的展示策略（`failed` 优先、按 `notAfter` 判过期、active 30 天内判 expiring），逻辑埋在 page 局部函数中。

## 问题证据和影响范围

- 证据：`src/pages/[locale]/certificates.tsx` 本地 `displayStatus` + `30 * DAY`
- 影响：仅 certificates 列表页；`domains/ssl` 仍使用 raw `certificate.status`（本轮不统一）

## 原始设计

页面内联：

```ts
function displayStatus(certificate, now) {
  if (status === "failed") return "failed";
  if (notAfter && notAfter <= now) return "expired";
  if (status === "active" && notAfter - now <= 30 * DAY) return "expiring";
  return status as DisplayStatus;
}
```

## 最小修改方案

- 新增 `src/lib/certificate-status.ts`：`certificateDisplayStatus` + `CERTIFICATE_EXPIRING_WINDOW_MS` + `CertificateDisplayStatus`
- 表驱动测试
- page 改 import，删除本地 DAY / type / function
- 不移动 `CertificateStatus` UI 组件；不改 `summary.active` 语义

## Design Challenger 结论

**APPROVE**

主收益是可测的非平凡时间策略，而非 call-site 数或未来 SSOT。禁止可配置 window、禁止接到 SSL 页。
