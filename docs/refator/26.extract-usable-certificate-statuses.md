# 26. Extract usableCertificateStatuses

## 发现的问题

runtime/nginx 引用证书时，4 处 worker 代码重复字面量 `["ready", "active"].includes(...)`。

## 问题证据和影响范围

1. `src/worker/lib/manager/root-input.ts` — manager TLS 绑路径
2. `src/worker/routes/settings.ts` — diagnostics（另加 notAfter 未过期）
3. `src/worker/lib/deployment/runner.ts` — publish/rebuild 证书资产可用性
4. `src/worker/lib/deployment/config-test.ts` — 同上

## 原始设计

各处内联同一状态集合。

## 最小修改方案

在 `src/shared/schemas/certificate.ts`（order status 常量旁）导出：

```ts
export const usableCertificateStatuses: string[] = ["ready", "active"];
```

4 处改为 `usableCertificateStatuses.includes(...)`。不增加 thin `isX` helper。settings 的 `notAfter` 条件保持独立。

## Design Challenger 结论

**REVISE** → 仅数组 SSOT，砍掉 thin isX → 按此实施

## TDD 过程

1. 常量契约测试（精确 `["ready","active"]`）
2. 导出常量并接线 4 处

- `src/shared/schemas/certificate-status.test.ts`（new）
