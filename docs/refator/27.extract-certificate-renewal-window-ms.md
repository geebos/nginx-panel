# 27. Extract certificateRenewalWindowMs

## 发现的问题

30 天证书窗口魔法值分散在 FE badge、renewal 调度、dashboard 指标与 ACME nextCheckAt 计算中，存在跨层漂移风险。

## 问题证据和影响范围

1. `src/lib/certificate-status.ts` — 原 `CERTIFICATE_EXPIRING_WINDOW_MS`
2. `src/worker/lib/acme/renewal.ts` — 本地 `renewalWindow` / 已 export `certificateRenewalWindowMs`
3. `src/worker/routes/dashboard.ts` — `renewalWindowEnd`
4. `src/worker/lib/acme/scheduler.ts` — `nextCheckAt = notAfter - 30d + jitter`

## 原始设计

多处独立 `30 * 24 * 60 * 60 * 1000`（或 FE 本地常量）。

## 最小修改方案

`src/shared/schemas/certificate.ts`：

```ts
export const certificateRenewalWindowMs = 30 * 24 * 60 * 60 * 1000;
```

四处接线；单一命名；`renewal.ts` re-export 保持兼容；删除 FE 双名。

## Design Challenger 结论

**APPROVE**（单名，勿保留 CERTIFICATE_EXPIRING_WINDOW_MS alias）

## TDD 过程

1. 既有 `certificate-status` 边界测试改用 shared 常量
2. 导出常量并接线；renewal 测试回归
