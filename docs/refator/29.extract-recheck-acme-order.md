# 29. Extract recheckAcmeOrder

## 发现的问题

域名 recheck 路由与 `recheckManagerOrder` 重复同一调度策略；非 recheckable 时 domain 响应缺 `debounced: false`，与 manager / FE 类型不一致。

## 问题证据和影响范围

- `src/worker/routes/certificates.ts` recheck handler
- `src/worker/lib/manager/certificate.ts` `recheckManagerOrder`
- FE `api.ts` 声明 `{ order; debounced: boolean }`

## 原始设计

两处各自实现 debounce / nextPollAt；domain 非 recheckable 省略 `debounced`。

## 最小修改方案

`src/worker/lib/acme/recheck-order.ts`：

- `recheckAcmeOrder(db, order) → { order, debounced }`
- 始终返回 boolean `debounced`（含 non-recheckable → false）
- 入口只做 load / 鉴权

## Design Challenger 结论

**APPROVE**（统一 `debounced` 为 additive 修复）

## TDD 过程

1. helper 单测：non-recheckable 不写库、schedule、debounce
2. 实现 + 接线；既有 certificates recheck 集成测回归
