# 30. Extract retryCloudflareOrderCleanup

## 发现的问题

域名 cleanup/retry 路由与 `retryManagerCleanup` 重复同一 Cloudflare 清理重试策略。

## 问题证据和影响范围

- `src/worker/routes/certificates.ts` `.../cleanup/retry`
- `src/worker/lib/manager/certificate.ts` `retryManagerCleanup`

## 原始设计

两处各自校验 cloudflare + terminal、写 pending、调用 cleanup、返回 publicOrder。

## 最小修改方案

`src/worker/lib/acme/retry-cleanup.ts`：

- `retryCloudflareOrderCleanup(db, order)`
- 入口只负责 load / 鉴权

## Design Challenger 结论

与 cancel/recheck 同一模式；多处重复策略 → **APPROVE**（本轮自审，与 28/29 先例一致）

## TDD 过程

1. 非 cloudflare / 非 terminal 拒绝
2. terminal cloudflare 路径 re-queue 后走 cleanup
3. 接线两入口

## 最终修改
