# 32. Extract retryAcmeOrderActivation

## 发现的问题

域名 activation/retry 与 `retryManagerActivation` 在 order 加载后重复：查 cert → activation → 缺则 409 → `retryCertificateActivation`。

## 问题证据和影响范围

- `src/worker/routes/certificates.ts` activation/retry
- `src/worker/lib/manager/certificate.ts` `retryManagerActivation`

## 原始设计

两处各自 resolve activation 再调用共享的 `retryCertificateActivation`。

## 最小修改方案

`src/worker/lib/acme/retry-activation.ts`：

- `retryAcmeOrderActivation(db, order, enqueue?)`
- 入口只负责 load/auth order

## Design Challenger 结论

与 cancel/recheck/cleanup 同模式 → **APPROVE**（自审）

## TDD 过程
